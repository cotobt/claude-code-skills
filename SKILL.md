# 世界杯竞彩预测 + 投注方案

## 核心哲学

不重复市场共识。目标是**找到市场低估的冷门价值**。

## 使用前提（重要）

**每次用户提及世界杯/竞彩/投注方案时，必须先加载本skill。**`skill_view(name='world-cup-predictor')` 然后跟着工作流程走。

不加载本skill的后果：
1. 缺少15维度评分系统，分析变相"凭感觉"
2. 缺少🚌冷门检测，错过核心价值
3. 缺少凯利公式和隐含概率提取，金额分配不合理
4. 缺少标准输出模板，每次产出格式不一致
5. 缺少木桶原则/单关规则等竞彩限制知识

如果因为没有先加载skill导致了非标准输出，用户需要重新解释需求，浪费双方时间。
产出顺序：冷门优先 → 简述热门 → 不出方案时不逐个列比赛。

## 工作流程

### 独立执行模式

用户指令决定执行范围：

| 用户指令 | 执行范围 | 输出要求 |
|:---|:---|:---|
| **"预测分析"、"分析"、"怎么看"** | Step 0→9（分析部分） | 不出方案，给出方向判断和规则触发 |
| **"出方案"、"投注方案"、"串关"、"买"** | 完整 Step 0→12 | 按标准方案模板输出 |
| **"回测"、"复盘"、"验证"、"历史"** | Step 0→9 + 回测模板 | 按 `references/backtest-template.md` 输出 |
| **"明日预测"、"世界杯比赛预测"**等模糊请求 | 默认完整 Step 0→12 | 按标准方案模板输出 |
| **用户贴出两个方案问"哪个更好"** | 方案对比评估流程 | 不走Step 1→9 |

**注意：** 用户明确说"预测分析"时，即使当天还有比赛未踢，也只做分析不出方案——用户将其视为独立步骤。

**回测模式补充：** 回测模式下不输出投注方案，重点验证 skill 核心规则的有效性。输出必须包含：
1. 回测概览
2. 场次清单与规则触发表
3. 规则触发统计表
4. 方向判断命中率表
5. 关键案例分析
6. 经验教训表
7. 局限性说明

详细格式参见 `references/backtest-template.md`。

## 使用流程（完整工作流）

**每次用户提出出方案需求时，按以下Step 0→1→2→...→12逐项执行，不可跳过。** 即使同一天同一批比赛已经分析过，也必须从Step 0重新开始，不使用任何历史对话中的数据和结论。

```
Step 0  加载skill → skill_view(name='world-cup-predictor')
Step 1  数据获取 → curl trade.500.com/jczq 抓赔率
Step 2  用户确认 → 系统列出场次清单，让用户确认
Step 3  首轮数据收集 → web_search搜战报，2个来源交叉确认
Step 4  15维度评分 → 15维度每队-2~+2，求净分
Step 5  市场-模型关系 → 方向冲突时跟市场走
Step 6  🚌冷门检测  → 传控vs大巴检测，含质量分级修正+延续修正
Step 7  隐含概率提取 → (1/赔率)/∑(1/赔率)
Step 8  模型概率（模型p）→ 市场p+评分净分映射，p×赔率必须>1.0
Step 9  价值过滤 → 模型p-市场p=空间，主攻需≥5%，填充≥3%
Step 10 空间权重预算分配 → 各场权重目标金额（非票面），必须出方案
Step 11 串关方案设计 → 组合排序+加注，预算转化为具体串关票
Step 12 产出方案 → 完整内容直接复制到对话框回复，按模板输出
```

### Step 0: 加载skill

`skill_view(name='world-cup-predictor')` — 这是入口检查点。**不加载不能往下走。** 每次新方案请求必须视为全新会话，从step1起全部重新执行。不得使用前次对话中的评分、赔率、模型结论。

### Step 1: 时间确认 + 数据获取

**⚠️ 时间确认检查（防日期混淆——犯过错的！）：**
在执行任何操作前，先 `date` 确认当前北京时间。然后根据用户请求的关键词区分：

| 用户说 | 含义 | 竞彩对应 |
|:------|:-----|:---------|
| "今天结束的" / "刚结束的" / "回测今天" | 前一个自然日的比赛（已完场）。例如当前6月21日18:00，则回测6月21日凌晨结束的比赛 | 需要查历史页面 |
| "明天的" / "6月23日的" | 往后一天的比赛（未踢或进行中） | 注意：500彩页面显示的是当前销售期，可能跨越两个自然日 |
| "回测6月22日" | 6月22日已结束的比赛（即6月23日凌晨前完场的） | 查500彩历史页面 |
| "给出XXX的预测分析" | 用户要的是提前预测分析，不是回测 | 加载skill，走完整Step 1→12 |

**常见陷阱（犯过错的）：**
1. **跨日混淆** — 用户说"明天的4场"，而500彩页面同时显示"周一041-044"（今天凌晨）和"周二045-048"（明天凌晨）。需要先date确认今天是周几，再根据500彩的matchnum前缀区分：`周一NNN` = 当天凌晨比赛，`周二NNN` = 次日凌晨比赛。
2. **"今天结束"含义** — 用户说"回测今天结束的四场"时，当前6月21日，那么6月21日凌晨的4场已经结束了，这就是"今天结束"的比赛。不要理解成"6月22日的比赛"。
3. **先date再判断，不凭记忆猜。**
4. **用户明确说"预测分析"时**，说明是提前预测而非回测，即使当天还有比赛未踢，也按预测不走回测流程。

**500彩matchnum前缀解读（抓取后的第一道判断）：**
先 `date` 确认今天是周几，然后看500彩返回的matchnum：
- `date` 输出 `Monday` → 500彩的 `周一NNN` = 今天凌晨/上午的比赛, `周二NNN` = 明天凌晨的比赛
- 用户说"明天的" → 如果当前是周一，而500彩显示`周一041-044`（今天凌晨）和`周二045-048`（明天凌晨），**需要问用户是指周一这几场还是周二的**，或者看用户之前提到的球队名称来判断。

**核心原则：根据用户请求的数据使用模式，选择合适的数据源。**

#### 模式判断

| 用户说 | 模式 | 数据源 |
|:------|:----:|:-------|
| "出方案"/"投注方案"/"串关"/"买" | 投注方案模式 | 只用 500.com 竞彩官方赔率 |
| "预测分析"/"怎么看"/"分析"/"预测" | 预测分析模式 | 允许多博彩机构/历史数据库 |
| "回测"/"复盘"/"验证"/"历史" | 回测模式 | 允许多博彩机构/历史数据库 |

**模式切换规则：**
- 用户明确说"出方案" → 必须进入**投注方案模式**，严格执行 500.com 单一数据源
- 用户明确说"预测分析" → 进入**预测分析模式**，可用多机构赔率做市场共识分析
- 用户明确说"回测" → 进入**回测模式**，可用历史数据库验证模型
- 用户表述模糊时，默认进入**投注方案模式**（最严格）

**方法1（投注方案模式首选）：500彩票网 — 竞彩足球让球胜平负页**
URL：`https://trade.500.com/jczq`
- curl抓取：`curl -s -L "https://trade.500.com/jczq/" | iconv -f gb2312 -t utf-8//IGNORE 2>/dev/null > /tmp/500.html`
- HTML中每场比赛的赔率在 data-sp 属性中：data-type="nspf"=胜平负，data-type="spf"=让球胜平负
- data-value="3"=主胜，1=平，0=客胜
- 页面为GB2312编码，注意iconv转码
- 用 Python re.findall 解析 `<tr class="bet-tb-tr">` 行

**提取规则：** 禁止用"约"字。**赔率汇总表必须用 `scripts/render-table.py` 的 `render_table()` 生成，禁止手敲表格。** Step 1的赔率表也不例外。

**⚠️ 投注方案模式数据完整性铁律（v3.2新增强化）：**
- 仅适用于"出方案"/"投注方案"/"串关"/"买"等请求。
- 竞彩页面某场比赛某一玩法显示"暂无数据"或data-sp为空 = 该玩法该场不可投注。**不能从其他页面、其他玩法、其他渠道自行补数据**。
- 例如：巴西vs海地在胜平负页显示无数据，就不能从其他页面的sp数组里扒数字来"补"胜平负赔率。那是编数据。
- 只认竞彩官方页面上该场比赛该玩法的可见赔率数据。不同页面的赔率数据不可串用。
- 不可投注的玩法在后续所有步骤中标注"不可投注"，仅分析可投注的玩法。
- **所有赔率必须来自同一页面同一玩法。** 同一场次不同玩法（胜平负vs让球胜平负）各管各的，不能混用。
- **⚠️ 出方案时只使用竞彩足球官方列出的可投注玩法。** 标准SPF缺失的场次，不能自己按让球盘折算一个"主胜≈XX"填进去。标准SPF缺失就用让球盘数据出方案，不能混用。
- **禁止使用非竞彩足球的赔率信息出方案**，包括但不限于：亚盘、欧赔、大小球、单场上盘下盘等。方案中的所有赔率必须来自竞彩足球页面。

#### 无赔率模式

**适用场景：**
- 回测历史比赛时缺少历史赔率
- 预测分析时某些比赛赔率尚未开出

**禁止：**
- ❌ 无赔率模式下禁止编造赔率出方案
- ❌ 无赔率模式下禁止计算具体 ROI
- ❌ 无赔率模式下禁止把定性分析伪装成定量方案

2. 这是同一系列连续请求（如用户在上一轮结束后接着问"那明天的"）
3. 用户使用了肯定语气（"再出个"、"给我出"），表明不需要先确认场次

跳过后，在方案开头用一句话列出所有比赛和对战关系，让用户快速验证。如果用户后续指出场次不对，回到Step 1重新抓取。

**不跳过的情况：** 用户只说"帮我看世界杯"、"有方案吗"等模糊请求——必须先列出所有比赛让用户确认选择哪些。

获取各队首轮战报数据：比分、控球率、射门/射正、关键事件。

- **首选操作：直接 `web_extract` 已知的权威中文新闻URL，一次性提取全部比分和赛况**，而不是用 `web_search` 到处搜。
- `web_search` 只用于不知道具体URL时定位页面。
- 中文权威源已验证可用列表（URL模式已知，可直接抓取）：
  - 新华网：`news.cn`（搜索"晨观世界杯"等专栏，通常一篇文章覆盖当天所有比赛）
  - 央视体育：`sports.cctv.com`
  - 新浪体育：`k.sina.cn`
  - 虎扑：`bbs.hupu.com`
  - 网易体育：`sports.163.com`
- **英文源（ESPN/BBC/Guardian等）网页大多被屏蔽无法抓取，不要浪费时间。** 先试中文源。
- 仍坚持**2个独立来源交叉确认**后方可确认数据。
- 中文搜索兜底关键词格式：`2026世界杯 [日期] 比赛结果 [球队1] [球队2] [球队3] [球队4]`

- 新闻标题可能是前瞻性预测不是最终赛果
- 只有在文章明确使用"完场""最终比分""赛果""full time""final"等用语时才确认比分
- 交叉验证至少2个独立来源一致才采用

### Step 3.5: 第二轮起必填检查表

从小组赛第二轮起，每场比赛前必须完成以下量化检查。**不可跳过。**

| 检查项 | 阈值 | 触发动作 |
|:-------|:-----|:---------|
| 射门转化率 < 5% | 进球/射门 | 🎈伪传控检测 |
| 射正率 < 30% | 射正/射门 | 冷门评分 +1 |
| 实际得分 vs xG偏离 > 1.5 | 进球 - xG | 运气修正：回归均值方向 +1 |
| 首轮控球 > 60% 却输球 | — | 标记为「控球陷阱队」 |
| 首轮控球 < 35% 却赢球 | — | 标记为「高效防反队」，🚌检测时提高权重 |
| **对手质量折扣：大胜鱼腩队的评分扣减** | 对手排名<100或新军 | D4评分最多+1（而非+2） |
| **热门方世界杯连续零封场次** | 连续≥5场未零封 | 冷门方进球概率加分+1 |
| **🐴黑马球队延续性** | 低排名方首轮平/胜强队 | 第二轮该队评分+1 |
| **🐴黑马球队退潮** | 连续2轮爆冷的大巴队 | 第三轮不可再追，回归正常评分 |

**对手质量折扣规则说明：**
- 当某队首轮大胜（≥4球差距），但对手是**世界杯新军或FIFA排名<100的鱼腩队**时，该队D4（近期战绩）评分**不得超过+1**。
- 原因：屠杀鱼腩的信息量≈0，不能当满分用。例：德国7-1库拉索（库拉索世界排名170+、世界杯新军）→ D4最多+1，不是+2。
- 例外：如果鱼腩队首轮前已被验证为强队（如非洲杯冠军、预选赛全胜等），不受此限制。

**热门方防线漏洞检查：**
- 热门方在世界杯赛场上连续≥5场未能零封 → 冷门方有望进球，评分+1
- 此信息在D8（打法风格）或D12（定位球效率）中体现均可。

**🐴 黑马球队标记规则：**

**规则1：黑马延续性（第二轮适用）**
- 条件：低排名球队（FIFA排名比对手低≥20位）在首轮逼平或击败强队
- 触发：该队第二轮比赛评分 +1
- 原因：首轮爆冷会提升士气、暴露对手弱点，市场可能仍低估该队
- 示例：摩洛哥首轮1-1平巴西 → 第二轮对苏格兰0-1取胜，黑马延续性+1正确

**规则2：黑马退潮（第三轮适用）**
- 条件：某大巴球队连续两轮爆冷（首轮+次轮均拿分或取胜）
- 触发：第三轮对阵该队时，**不可再追其爆冷方向**，应回归正常实力判断
- 原因：连续爆冷后市场会重新定价，对手也会针对性布置；大巴队体能和运气难以持续三轮
- 示例：佛得角首轮0-0西班牙、次轮2-2乌拉圭 → 第三轮不可再追佛得角不败

**规则3：黑马甄别（排除伪黑马）**
- 以下情况不标记为黑马：
  - 靠点球/红牌/乌龙等偶然因素赢球
  - 对手严重轮换或轻敌
  - 全场被围攻但靠唯一反击赢球（xG严重落后）
- 例：德国世界杯连续7场丢球（上一次零封是2014年决赛）→ 科特迪瓦有望进球，冷门评分+1

**回测验证：** 土耳其第一轮0-2输澳大利亚时已暴露终结问题（30射0球）。如果第二轮前强制填表，不会漏掉伪传控预警。

| 6 | 人员可用性（核心缺阵=-2） |
| 7 | 主教练（>3年+>60%=+2） |
| 8 | **打法风格**（🚌检测基础） |
| 9 | 阵容默契度 |

**D10 旅途 & 场地（含东道主加成）：**

| 条件 | 分值 | 说明 |
|:-----|:----:|:-----|
| 东道主球队在本土/联合举办国比赛 | +1 | 2026美加墨联合举办，东道主在三国境内比赛均适用 |

| 高原/极端气候不适应 | -0.5 | 如墨西哥城海拔2240米 |

**东道主量化规则：**
- 东道主球队 D10 最高 +1，**不可超过 +1**
- 东道主优势要结合首轮表现校准：
  - 若东道主首轮大胜鱼腩，按对手质量折扣处理，不追加加分
  - 若东道主首轮被爆冷（如加拿大1-1波黑），需审视真实实力，但第二轮仍保留+1基础分
- 联合举办国之间流动参赛，仍视为东道主球队
- 东道主不是稳胆，只是加分项

**回测验证：**
- 墨西哥（东道主）：2-0南非、1-0韩国 → 东道主加成合理
- 美国（东道主）：4-1巴拉圭、2-0澳大利亚 → 东道主加成合理
- 加拿大（东道主）：1-1波黑、6-0卡塔尔 → 首轮被低估，次轮反弹，东道主基础分+1有效

**D2 战意量化表：**

| 形势 | 分值 |
|:-----|:----:|
| 输球=淘汰 | +2 |
| 输球=丧失主动权（需看别人脸色） | +1 |
| 平局即可出线 | -1（保守策略） |
| 已出线+可能轮换 | -2 |
| 已淘汰+荣誉战 | ±0（不可预测） |
| 两连败已淘汰 | -1（士气崩盘） |

**第三轮小组赛战意特殊规则：**

第三轮小组赛战意最复杂，必须结合积分形势判断，不能只看球队实力：

| 场景 | 两队战意 | 比赛风格预测 | D2调整 |
|:---|:---|:---|:---:|
| A队需胜，B队已出线 | A +2，B -2 | A强攻，B轮换/保守 | A+2，B-2 |
| A队需胜，B队已淘汰 | A +2，B ±0 | A强攻，B荣誉战不可预测 | A+2，B±0 |
| A队平局即可出线，B队需胜 | A -1，B +2 | A保守，B强攻 | A-1，B+2 |
| 两队平局即可携手出线 | A -1，B -1 | 默契球/保守 | 双方-1 |
| 两队都需胜才能出线 | A +2，B +2 | 对攻，大球概率高 | 双方+2 |
| 两队都已淘汰 | A ±0，B ±0 | 荣誉战，不可预测 | 双方±0 |
| 一队已出线但争小组第一 | +1 | 仍想赢但可能留力 | +1（非+2） |

**第三轮关键原则：**
1. **先看积分表，再分析比赛**：脱离积分形势的战意判断都是拍脑袋
2. **已出线球队不等于放水**：要区分"已稳出线"和"争小组第一"，后者仍+1
3. **平局双出线场景要警惕默契球**：双方都可能保守，强队也未必赢
4. **荣誉战不可预测性高**：已淘汰球队可能爆冷也可能崩盘，不宜重注
5. **淘汰边缘球队强攻暴露空档**：落后时大举压上，反而容易被反击

**示例（假设）：**
- 西班牙4分、日本4分、哥斯达黎加3分，第三轮西班牙vs日本：双方平局即出线 → 双方D2 -1，比赛可能保守
- 德国1分、哥斯达黎加3分，德国必须赢 → 德国D2 +2，哥斯达黎加可接受平局 → 哥斯达黎加D2 -1

**D3 排名差+首轮结果交叉校准：**
- 排名差≥25位 AND 低排名方首轮面对强敌拿分（平/胜）→ 低排名方评分+2
- 回测验证：摩洛哥排名7vs苏格兰42（差35位）+ 摩洛哥首轮1-1平巴西 → +2分正确捕获冷门

### Step 5: 市场-模型关系

- **冲突覆盖**：非市场维度方向 ≠ 赔率方向 → 跟市场走，降置信度
- **置信度天花板**：市场conviction < 50% → 不能输出High
- **负向堆叠放大**：≥3维度同时-2 → 额外扣3分

### Step 6: 🚌 冷门检测机制（核心）

**触发条件（四条必须都满足，不能用结果反推）：**

1. **热门方是否为传控/高压队**（满足≥2条才确认）：
   - 控球率 ≥ 55%
   - 打法含 possession / build-up / high press / gegenpressing
   - 场均射门 > 15
   - 教练为传控或高压流
   - ⚠️ **纯高压（仅gegenpressing/high_press但无possession/build-up/tiki-taka）不视为🚌传控类型**

2. **冷门方是否大巴队**（控球≤42% / low block / 5-4-1），且排名差≥20位
3. **热门方不是东道主**
4. **排名差 ≥ 20 位**

**触发后：** 走「让球胜平负: 让负」，不买热门方胜平负。

**🚌强度修正（连乘链算法）：**

🚌最终分 = 3（触发基础分）× 各修正系数。评分解读：0-1无冷门 / 2-3让负有价值 / 4-5让负/平局 / 6-10重注冷门。

| 条件 | 系数 | 理由 |
|------|:----:|------|
| 热门有精英空中支点（Haaland/Kane/Lukaku/195cm+高中锋） | ×**0.5** | 传中砸头球破大巴最有效 |
| 热门**没有**精英空中支点 | ×**0.9** | 缺少空中手段 |
| 冷门方有精英门将（近期神扑/零封记录） | ×**1.3** | 门神加成 |
| 热门传控质量**高效**（控球≥60%+射正≥8+预期进球≥1.5） | ×**0.5** | 高效传控破大巴能力强 |
| 热门传控质量**中效**（控球≥60%+射正5-7+预期进球1.0-1.49） | ×**0.7** | 中效传控，适度折半 |
| 热门传控质量**低效**（控球≥60%+射正<5+预期进球<1.0） | ×**1.0** | 低效传控，不调整 |
| 热门进攻缺组织核心 | ×**0.8** | 缺少喂球者 |
| 热门阵容磨合生涩（新帅/新体系） | ×**0.8** | 磨合问题 |
| 热门赔率 < 1.40 | ❌**忽略** | 实力差过大 |

### 🎈 伪传控陷阱检测

当热门方满足以下≥2条时触发。**分两套阈值：单轮检测（首轮后）和跨轮检测（第二轮起）。**

**单轮检测（首轮后适用）：**

| 条件 | 阈值 | 说明 |
|:----|:-----|:-----|
| xG - 实际进球 > 2.0 | 预期进球vs实际进球偏离 | 极端转化崩溃，比单纯射门数更稳定 |
| 运动战进球 = 0 | 不含点球 | 点球会刷高进球数但掩盖运动战无力 |
| 射正率 < 25% | 射正/射门（≥8次射门样本） | 每4脚射门不到1脚中目标 |

**首轮低效率预警（单轮即可触发，黄色预警）：**
即使单轮检测3条未全部满足，若出现以下情况，仍应标记为🎈低风险：
- 控球率 ≥ 55% AND 射门 ≥ 15次 AND 射正率 < 30%
- 控球率 ≥ 60% AND 射正 < 5次
- xG ≥ 1.5 AND 实际进球 = 0

黄色预警触发后：
- 热门方D8（打法风格）扣0.5分
- 冷门评分+1
- 下一场比赛前升级为重点关注对象

**跨轮检测（第二轮起适用，补充检测）：**
即使单轮检测未触发，如果某队满足以下任一条件，也触发🎈警告：
- **连续两场比赛进球=0**（首轮+次轮）
- **连续两场比赛射正率<25%**（每场射门≥8次样本）

跨轮检测触发后：
- 该队D4（近期战绩）扣1分——连续低效不是运气问题
- 冷门评分+1
- 标记为「慢性锋无力」

**示例（厄瓜多尔 vs 库拉索 2026-06-21）：**
- 首轮0-1科特迪瓦（全场仅1射正），次轮0-0库拉索（全场3中框）
- 单轮xG偏离未触发（首轮xG低因3中框但预期进球不高）
- **跨轮检测：连续2场进球=0 → 🎈慢性锋无力触发**
- D4扣1分，冷门评分+1

**不包含的阈值（已评估但否决）：**
- ~~首轮射门≥25次~~ → 30射0球样本太少，下一届可能遇不到第二次
- ~~头号射手身价<2000万€~~ → 身价信息获取成本高，且与xG高度相关

**触发后：**
- 热门方D8（打法风格）扣1分——传控是假的，效率是假的
- 冷门评分自动+2
- 推荐：冷门方向（让胜/让负），不追热门穿盘

**示例（土耳其 vs 巴拉圭 2026-06-20）：**
- 首轮xG≈2.5+，实际进球0 → 偏离>2.0 ✓
- 运动战进球=0 ✓
- 射正率：首轮30射？射正（待查），第二轮13射2射正=15.4%<25%
- **3/3触发 → 🎈命中**

### 🎈 降权因子双向化（建议2 P0）

🚌修正表中的以下规则：

| 条件 | 系数 | 当前用途 |
|:----|:---:|:---------|
| 热门传控质量**低效**（控球≥60%+射正<5+预期进球<1.0） | ×**1.0** | 仅做🚌降权 |

改为：
- **仍作为🚌降权因子**（不变）
- **同时作为独立冷门加分项**：热门首轮射正<6 AND 首轮进球≤1 → 冷门评分+2
- 这个改动独立于🚌是否触发，即🚌未触发时仍可加分

### 情景大巴检测（建议5 P1）

第二轮起适用：

```
IF 冷门方 首轮曾率先进球（领先过）
AND 首轮被射门数 > 15（领先后被围攻）
→ 标记为「情景大巴队」
→ 🚌检测中按大巴处理，但附加标注「情景型」
```

**回测验证：** 巴拉圭打土耳其64秒进球+领先后主动回收，属于情景大巴。当前🚌检测中冷门方首轮控球>40%会判定不适用，情景大巴规则补上了这个缺口——领先后控球率数据会被稀释，只看全场控球会漏判。

**计算示例（巴西 vs 海地 2026-06-20）：**
- 基础分：3（🚌触发）
- 传控中效（5射正）：×0.7 → 2.1
- 无精英高中锋：×0.9 → 1.89
- 缺组织核：×0.8 → 1.51
- 赔率1.65>1.40：不忽略
- **最终🚌分=1.51** → 0-1区间→无冷门

**🚌延续修正：**
- 某队上一轮通过🚌方向命中 → 当前冷门评分-3
- 某队上一轮被🚌冷门方击败 → 当前让球盘评分+1
- 某队上一轮🚌未触发但爆冷 → 不调整

**特别规则：**
- 热门赔率<1.40 → 🚌忽略
- 热门是东道主 → 🚌评分-2
- 强强对话（FIFA差<15且双方排名<20）→ 🚌不适用

### Step 7: 隐含概率提取

```
真实概率 = (1/赔率) / (1/主胜 + 1/平 + 1/客胜)
```

列出每场比赛所有可用选项的赔率、市场p和空间。与价值过滤（Step 9）合并为一次计算，不重复。

### Step 8: 模型概率（模型p）确定

**⚠️ 模型p的确定方法（v3.2新增，防止凯利负数bug）：**

**模型p = 市场p + 评分净分映射加成**。不要单独从泊松λ修正推导模型p——之前的方案从未出现凯利负数问题，说明泊松λ法容易算错。

评分净分映射表（基于2026世界杯6月12日-6月22日回测数据优化）：

| 评分净差范围 | 模型p加成 | 适用场景 |
|:-----------:|:---------:|:---------|
| 0~5 | 市场p + 0.5~1.5% | 微弱优势，基本跟随市场 |
| 6~10 | 市场p + 2~3.5% | 有一定优势但不算强烈 |
| 11~15 | 市场p + 4~6% | 明显优势，值得重视 |
| 16~20 | 市场p + 6.5~9% | 强烈信号，如美国+19→+6.9% |
| 21~25 | 市场p + 9.5~12% | 非常强信号 |
| 26+ | 市场p + 12~15% | 碾压级优势，极少出现 |

**赔率区间修正（v3.13.0新增）：**

评分净分映射加成不是一成不变的，需要根据热门方赔率进行修正：

| 热门方赔率区间 | 修正系数 | 原因 |
|:-------------:|:-------:|:-----|
| < 1.60 | ×0.7 | 市场对强队定价通常较准，需要更强证据才偏离市场 |
| 1.60 ~ 2.50 | ×1.0 | 正常应用基础映射表 |
| > 2.50 或判断为冷门方 | ×1.2 | 市场对冷门/均势局定价偏差较大，模型优势可放大 |

**修正示例：**
- 评分净差+18，热门方赔率1.40 → 加成 = 8% × 0.7 = 5.6%
- 评分净差+12，热门方赔率2.10 → 加成 = 5% × 1.0 = 5%
- 评分净差+8，判断冷门方方向，赔率3.20 → 加成 = 3% × 1.2 = 3.6%

**调整原因（基于回测）：**
- 低赔率强队（如德国1.36、阿根廷1.30）市场定价精度高，评分加成过大容易高估
- 中赔率区间（1.60-2.50）是冷门价值最集中的区域，基础映射表直接适用
- 高赔率冷门方向市场容易低估，适当放大加成有利于捕捉价值

**绝对检查：** 模型p × 赔率 必须 > 1.0。如果 < 1.0，按以下清单逐项核查评分是否算错：

**模型p×赔率<1.0核查清单：**
| 检查项 | 说明 |
|:-------|:-----|
| D4对手质量折扣是否漏扣？ | 大胜鱼腩队(排名<100/新军)只给+1不是+2 |
| 🔒防线漏洞是否漏查？ | 热门方连续≥5场世界杯未零封→冷门加分+1 |
| 🐟冷门方预选赛/杯赛防线记录是否漏查？ | 如科特迪瓦预选赛10场零失球→D12加分 |
| 🎈伪传控是否漏检？ | xG偏离>2.0+运动战进球=0+射正率<25% |
| D2战意是否误判？ | 加分规则有标定时用量化表，不靠拍脑袋 |
| 评分方向是否标反？ | 确认净差正负号含义：正值=左侧球队强，负值=右侧球队强 |

如果确认评分无误，仍按模型p出方案，凯利金额取最小值¥2。**不能因为凯利负值就跳过不出方案**——用户给了预算就必须出方案。

### Step 9: 价值过滤

- **主攻腿**（2串1核心）：空间 < 5% → **不投**
- **串关填充腿**（赔率≤2.00且模型p≥55%）：空间3-5% → 可接受

价值空间 = 模型p - 真实概率

### Step 10: 凯利公式（按空间占比分配预算）

核心逻辑：**按各选项的空间占比分配总预算**，不是用半凯利f公式逐场计算。

```
总空间 = 所有场次空间之和
某场权重 = 该场空间 / 总空间
该场权重目标金额 = 总预算 × 该场权重 → 舍入到 ¥2 倍数
```

**规则：**
- 空间 = 模型p - 市场p（即Step 9算出的价值空间）
- 空间越大，该场的权重目标金额越大
- 金额舍入到¥2倍数，最低¥2
- 调整超预算或不足时：从空间最低的场次开始减/加

**Step 10产出的不是票面金额，而是各场的权重目标。** 具体如何分配到串关票面上是Step 11的工作。
- **用户给了预算就必须出方案。** 不能因为凯利f≤0、或只有1场通过价值过滤、或比赛非单关，就说"没有可投选项"跳过。
- 当f≤0时：金额取最小值¥2（或预算的10%，取大值）
- 当金额舍入后<¥2时：取¥2
- 按用户的预算走，不用自己的判断替用户决定投不投
- 如果只有1场比赛且非单关，仍然出方案，标注"非单关，需走线下/其他渠道"或按用户要求的方式处理

**金额按凯利权重分配：space越大金额越大。** 不搞平均分配。

### Step 11: 竞彩串关方案

**核心规则：**
- 单关：胜平负/让球大多不开单关，全部走串关
- 混合过关：不同场可混玩法，同场不可混
- **复式投注（同一场多选）：** 可对同一场比赛的胜/平/负选2个或3个结果。

### ⚽ 让球胜平负基础（重要——必须理解，出过错的！）

**让球就是把让的球数加到被让方的比分上，重新计算比分来决定让球胜平负的结果。**

也可以理解为：开球时让球方先让出N球（0-N落后），然后看全场结束后的虚拟比分。

**示例（德国让-1球 vs 科特迪瓦）：**
```
开场虚拟比分：德国 0-1 科特迪瓦
全场实际比分：德国 2-1 科特迪瓦
让球后虚拟比分：德国 1-1 科特迪瓦 → 让平
```

**三种结果判定：**
| 实际赛果 | 让球后虚拟比分 | 让球胜平负结果 |
|:--------|:--------------|:-------------|
| 德国赢2球+（3-0、4-1） | 2-0、3-1 | **让胜** |
| 德国赢恰好1球（2-1、1-0） | 1-1、0-0 | **让平** |
| 德国平/输（1-1、0-1） | 0-1、-1-1 | **让负** |

**注意：让负 ≠ "冷门方方向"**
- 让负的范围是热门方平或输球
- 热门方赢恰好1球是让平，不是让负
- 以前犯过错：把德国2-1科特迪瓦算成让负命中，实际是让平

**同样，让球SPF的含义：**
- 500彩票页面上的spf（让球胜平负）赔率：
  - `data-value="3"` = 让球后主队胜（**让胜**）
  - `data-value="1"` = 让球后平局（**让平**）
  - `data-value="0"` = 让球后客队胜（**让负**）
- 而nspf（标准胜平负）的data-value含义相同，但不涉及让球。
- **竞彩让球胜平负中，"胜"指"让胜"，"负"指"让负"，"平"指"让平"。** 不要在方案描述中使用歧义表述。正确示例：`德国让负(科+1)`、`德国让平`、`荷兰让负`。

### 🧠 从判断到选项的映射规则（重要——犯过错的！）

**核心原则：你的盘口判断 ≠ 你该选的竞彩选项。**

把"判断"映射到"选项"时，必须逐条核对以下规则，**不能跳过"判断→选项"的翻译步骤直接选**。

**规则1：判断"某队赢" → 选SPF胜（不让球），不是让球胜平负**
- 正确：判断"比利时赢" → 比利时胜(SPF) @1.32
- 错误：判断"比利时赢" → 比利时让负(-1) @2.94（让负命中条件是比利时不胜，跟判断完全相反）
- SPF胜覆盖所有赢球场景（赢1球、2球、3球都收），是"某队赢"最诚实的映射

**规则2：判断"某队赢但不会大胜/穿盘" → 仍然是SPF胜**
- 盘口判断（"不会穿-1"）和选项选择（"选让负"）是两回事
- 选SPF胜=覆盖所有赢球，选让负=赌不胜，两者不能混淆
- 如果既不想赌穿盘又不想赌不胜，可以选**让平**（赢恰好1球），但命中面窄

**规则3：只有判断"某队赢不了"时才选让负**
- 让负的命中条件是热门方平或输
- 适用场景：🚌触发后推荐让负（因为热门方传控效率低，冷门方大巴能守住）
- 如果你自己认为热门方会赢，就不要选让负——不管你是否认为它会穿盘

**规则4：判断"某队会大胜" → 选让胜**
- 让胜的命中条件是热门方赢2球+
- 适用场景：实力碾压+状态好+对手弱
- 注意搭配对手质量折扣规则（🐟大胜鱼腩时要打折扣）

**规则5：标准SPF和让球SPF互不通用**
- 同一场比赛的标准SPF和让球SPF是两个独立玩法
- 不能因为标准SPF赔率低就转去让球盘找高赔——那是不同的命中条件
- 出方案时每场比赛只选一个玩法（要么SPF要么让球SPF），同场不可混

**验证清单（每次出方案前过一遍）：**
```
□ 我选的选项的命中条件是什么？能说清楚吗？
□ 这个命中条件跟我自己的判断一致吗？
□ 让负=不胜，让平=赢1球，让胜=赢2球+，SPF胜=赢球
□ 如果我判断"某队赢"，我选的是SPF胜还是让负？
   - 选SPF胜 ✅
   - 选让负 ❌（跟判断相反，除非你有额外理由）
□ 如果我选了让负，我的真实判断是"这支球队赢不了"吗？
```

### 让球盘深度感知子规则（用于评分维度15）：

- 模型看好主胜，但主队让球≥0.5/1（需赢2球）且排名差<30 → 主队得分-2
- 模型看好主胜，但主队让球≥0.5/1且排名差≥30 → 正常，不扣分
- 让球盘阈值与实际赢盘率不匹配（赔率<1.50但近5场赢盘率<40%）→ 主队额外-2分

**本质理解：复式=在同一张串关票上对某场比赛选多个结果，每个结果组合对应1注独立注单。** 选2个结果=在该场比赛上有2种可能，与其他场的选项组合后生成多注独立注单。

**例：票"美国主胜 × 摩洛哥胜/平 × 巴西让胜 × 土耳其胜/平，3倍"**
- 这场比赛有4注独立注单（摩洛哥2选 × 土耳其2选 = 4种组合）
- 各注相互独立，每注对应1个具体的4场结果组合
- 比赛结束后，摩洛哥和土耳其各只有1个实际结果，因此**4注中最多只中1注**，其余3注作废
- 不存在"中多注"的情况

**彩票最高奖金：** 彩票票面上的"最高奖金"按4注中赔率最高的那注计算（单注赔率×¥2×倍数），不是4注合计。
    
**注数计算：** 
- 1场比赛复式（选2个结果）：注数×2
- 2场比赛复式（各选2个结果）：注数×2×2=×4
- 每注¥2，倍数统一加在每注上

**中奖规则：** 同一场比赛只有1个结果，复式中最多只有1注中奖。按实际结果对应的那注赔率兑付。

**场景分析中的处理：** 复式方案的场景分析只列**实际可能的中奖情况**（N种）。例如2场复式（各2选）只有4种中奖情况，每种写一行，回报=该注赔率×倍数金额。不列"中多注"场景——因为不存在。
- 木桶原则：纯胜平负/让球最高8关

**复式触发条件（Step 9→Step 11之间判断）：**

**规则A — 空间不足覆盖（原规则）：**

| 情况 | 用哪套 |
|:----|:------|
| 4场中没有任何一场空间在2-4%区间 | 原11注方案（全单选） |
| 4场中恰好有1场空间在2-4%区间 | 对该场做复式，其余单选 |
| 有≥2场空间在2-4%区间 | 选空间最高的那条做复式，其余单选 |

**规则B — 核心腿防穿覆盖（新增，优先级高于规则A）：**

当核心腿存在以下特征时，无论价值空间多少，都应对核心腿做复式覆盖：

| 特征 | 说明 | 示例 |
|:----|:-----|:-----|
| 🚌触发的核心腿 | 热门方传控vs冷门方大巴，推荐让负方向。但热门方实力碾压，也有让胜可能 | 西班牙让-2vs沙特：让负(3.06)是🚌推荐，但让胜(1.80)可能性不低，复式覆盖让胜+让负 |
| 核心腿赔率悬殊 | 核心腿赔率≥2.50，反方向赔率≤2.00 | 让负3.06 + 让胜1.80 → 两个方向都有合理性 |
| 核心腿方向存在明确反例证据 | 首轮数据支持两个方向（如首轮27射0球但xG高表明机会在） | 西班牙27射0球→该让负；但xG2.5+→该让胜 |

**触发后：**
- 核心腿做复式覆盖（两个方向各1注，基础2注=¥4）
- 复式部分只排除中间结果（让球盘=让平，标准盘=平局）
- **复式分配原则：主方向（空间大的）配重仓，反方向配轻仓。** 如西班牙让负3.06是主方向，让胜1.80是反方向轻仓覆盖。
- 同时设置一条**不含核心腿的独立串关票**作为全灭兜底

**规则C — 全灭兜底票（新增，任何时候都适用）：**

**每条方案必须至少包含一张不含核心腿的串关票。**

原因：核心腿如果全错（穿盘或中间结果），含核心腿的票全部归零。如果没有兜底票，方案就是"核心腿中→赚，核心腿错→全灭"的二元结果，容错为零。

兜底票设计原则：
- 不含核心腿，由其余场次中空间最高的2-3条腿组成
- 金额占预算的15-25%（¥6-¥10左右）
- 如果只有2条填充腿，直接做2串1
- 如果有3条填充腿，可以做3串1（容错1场）
- **兜底票的核心价值不是高回报，而是核心腿错时仍有回款，不让用户全灭**

**⚠️ 复式规则B触发提醒（犯过错的——方案初版漏掉复式，被用户指出）：**
当核心腿是通过🎈伪传控检测选出的冷门方向（让负），且热门方是实力碾压级球队（FIFA排名前10 vs 排名50+），**必须主动检查规则B**：

- 热门方首轮xG>2.0但进球=0（机会在，回归均值可能导致大胜）
- 冷门方首轮面对的锋无力对手（如沙特1-1乌拉圭—乌拉圭27射但锋无力≠沙特防线固若金汤）
- 核心腿赔率≥2.50，反方向赔率≤2.00

这三个条件满足任意2条 → 规则B触发。**不要等用户说"为什么不复式"才想起来！**

**三规则优先级：规则B > 规则A（同时触发时取规则B），规则C始终附加执行。**

**2-4%区间的含义：** 空间在2-4%意味着模型略看好但信心不足（低于5%的主攻门槛，但高于1-2%的纯填充）。此时单选该方向风险较高，用胜/平复式覆盖可提高容错。

**复式方案的注数自动计算规则（不是固定模板）：**
1. 先按正常流程生成所有串关组合（6×2串1 + 4×3串1 + 1×4串1 = 11注）
2. 遍历所有组合，**如果某组合包含被选为复式的场次**，生成两张独立票：复式版（胜/平各1注=¥4）+ 加注版（选空间更高的方向加注）
3. 不含复式场的组合保持单选（1注）
4. 加注规则不变：空间最高的组合（含核心腿的）可在基础注数上加倍数

**复式合并规则（出票时优化展示）：**
复式场的票应合并为一张票展示，而不是拆成两张列出。合并方式：
- **复式基础版**：方向覆盖（胜/平），1倍×2注=¥4
- **加注版**：单选空间较高的方向（如土耳其胜），加倍数（2倍×1注=¥4）
- 这样比拆开写（土耳其胜¥6+土耳其平¥2）更清晰，用户可以直观看到"复式覆盖+方向偏重"

**示例（土耳其复式+加注）：**
```
票3 | 2串1 | 美国主胜 × 土耳其胜/平 | 2注 | 1倍 | ¥4（复式）
票4 | 2串1 | 美国主胜 × 土耳其胜 | 1注 | 2倍 | ¥4（加注）
```
而不是：
```
票3 | 2串1 | 美国主胜 × 土耳其胜 | 3倍 | ¥6
票4 | 2串1 | 美国主胜 × 土耳其平 | 1倍 | ¥2
```

**预算超支修剪：** 当复式导致预算超支时，优先移除**不含核心腿的复式组合**（如摩洛哥×土耳其、巴西×土耳其），保留含核心腿的所有组合。见 `references/fushi-budget-trimming.md`
- 每注¥2，金额必须是¥2倍数
- 无防御组合、无反向选项
- 不搞重复注

**Step 10→Step 11的金额分配逻辑：**
Step 10产出各场的权重目标金额（按空间占比分配预算）。Step 11将这些权重目标转化为串关票面：
1. 先生成所有可能的串关组合（2串1+3串1+4串1共11注）
2. 按组合的总空间（组合内各场空间之和）从高到低排序
3. 排序靠前的组合加注（加倍数），排序靠后的保持基础¥2
4. 直到总金额用满预算


### 场景分析必验证项（防错清单）——每次出方案前逐项检查，少一项不能输出

- 场景分析命中情况列**必须用球队名**（如"仅美国错""仅摩洛哥错"），禁止用"仅主胜错"等选项名——同一选项名会出现在多场中，混淆用户
- 场景分析票号必须用程序循环计算，禁用人工填写/硬编码。这是有血泪教训的——手写"仅苏格兰错=2369"但实际正确是456，导致输出错误。
- **场景分析中复式场的命中条件必须逐注独立验证。** 每注的每个选项都必须满足才能中奖。例如：注"美国胜×摩洛哥胜×巴西胜×土耳其胜"——如果摩洛哥负（输球），该注不中，无论土耳其是否胜。**不要因为土耳其胜就认为包含土耳其胜的注都能中**，必须逐场检查。这是有血泪教训的——把"摩洛哥负+土耳其胜"算成了¥38，实际应为¥0。详见 `references/scenario-verification-pitfalls.md`。
- **彩票奖金计算：** 彩票票面显示的"最高奖金"是**单注最高奖金**（按最高赔率的那注×倍数×¥2计算），不是所有注相加的总和。实际中奖时，如果多注同时命中，奖金=各注中奖金额相加。场景分析应展示**实际中奖金额（各注相加）**，而非票面最高奖金。详见 `references/scenario-verification-pitfalls.md`。
- **场景表中"命中情况"列必须用球队名代替选项名**，防止选项名重复导致歧义（如"主胜"出现两次）。参考 `references/output-format.md`。格式：`仅[球队名]错` / `[球队1]+[球队2]`
- 具体做法：定义每张票的选项映射，遍历场景中的命中/未命中列表，对每张票检查其全部选项是否都在命中集合中。用代码算，别用脑子想。

3. **给出我的综合判断**：在对比后明确说倾向于哪个方案，以及为什么

## 淘汰赛特殊规则

世界杯淘汰赛阶段（1/16决赛、1/8决赛、1/4决赛、半决赛、决赛）与小组赛逻辑不同，必须单独处理。

### 淘汰赛核心差异

| 差异点 | 小组赛 | 淘汰赛 |
|:---|:---|:---|
| 平局处理 | 各拿1分 | 90分钟平局→加时→点球 |
| 战意 | 可能保留实力 | 必须全力以赴 |
| 冷门概率 | 较高（有默契球/轮换） | 较低，但加时/点球有偶然性 |
| 竞彩结算 | 90分钟结果 | **只结算90分钟结果** |
| 强队落后 | 可接受平局 | 必须强攻，防守空档大 |

### 淘汰赛规则调整

**1. 战意判断简化**
- 淘汰赛默认双方 D2（战意）均为 +2
- 不存在"已出线轮换""荣誉战"等复杂情况
- 只有伤病/停赛才会影响战意

**2. 竞彩只算90分钟**
- 投注方案中的胜平负、让球胜平负都只结算90分钟（含补时）
- **加时赛和点球大战不计入竞彩结果**
- 强队90分钟内被爆冷 ≠ 最终被淘汰

**3. 落后强攻场景**
- 如果模型判断强队可能落后或平局进入加时：
  - 强队会压上进攻，反击空间增大
  - 下半场进球概率提高
  - 但90分钟内穿盘难度增大

**4. 点球大战预判（仅供参考，不出方案）**
- 点球大战不纳入竞彩投注
- 可作为风险提示：门将点球扑救能力、历史点球大战表现

**5. 🚌冷门检测调整**
- 淘汰赛冷门方多为大巴队死守120分钟
- 但强队求胜欲望强，🚌触发后仍建议谨慎
- 排名差极大的比赛（如巴西vs弱队），即使🚌触发，也要考虑加时/点球风险

**6. 比分倾向**
- 淘汰赛常规时间平局概率高于小组赛
- 强队小胜（1-0、2-1）概率高
- 大比分屠杀概率低

### 淘汰赛投注建议

- **优先看让球盘**：淘汰赛强队让球较深时，让平/让负价值可能更高
- **警惕"必胜局"陷阱**：强队90分钟内未必能赢，但竞彩只算90分钟
- **关注体能和伤病**：淘汰赛强度大，体能和伤病影响更明显

**新增规则：**

### 「排名差+首轮结果」交叉校准

排名差≥25位 + 低排名方首轮面对强敌拿分（平/胜）。

**触发后加分需同时满足以下约束（防误报）：**

| 约束 | 原因 |
|:----|:-----|
| 对手无核心缺阵 | 对手缺核心时拿分含金量打折，例如巴西首轮缺内马尔 |
| 低排名方首轮xG不显著落后 | xG大幅落后说明拿分靠运气，不可持续 |
| 两项都满足 | 低排名方评分+2 |
