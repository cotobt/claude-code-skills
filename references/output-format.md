# 输出格式规范

## 核心规则

1. **所有表格必须用 `render_table()` 函数生成**
   - 不手打任何表格线
   - 函数定义固化在skill SKILL.md中
   - 每次产出方案时在execute_code中定义该函数，用实际数据调用

2. **输出顺序**
   - 方案标题（含预算）
   - 模型分析摘要表 → 逐场简述 → 投注明细表 → 出票校验表 → 场景分析表
   - 不用问"要不要发飞书"，终端输出即可

3. **⚠️ 方案交付铁律：出方案脚本运行后，必须将脚本产出的完整内容（标题、模型分析、投注明细、场景分析、免责声明）直接复制到对话框回复中。不允许只回复一句"方案已输出"然后让用户去看terminal输出。用户看不到terminal的脚本输出。**

## 模板

### 标题
```
💰 竞彩投注方案 | [日期]世界杯[N]场 | ¥[预算]
```

### 模型分析摘要表
表头：编号 | 对阵 | 玩法 | 选项 | 赔率 | 模型p | 市场p | 空间 | 结论
结论列值：⭐锚 / 🎯隐藏 / 🔥核心 / 填充 / ⚠️搏冷 / ❌淘汰

### 逐场简述
- 每场一行，格式：`▎ [编号]：判断理由`
- 🚌触发的场次用 ⚠️ 标注

### 比赛预测分析（放在模型分析摘要表和投注明细表之间）

每场比赛一段独立分析，格式：
```
▎ [编号] [主队] vs [客队] | [时间] | [组别]
[首轮战绩简述，关键数据（控球率、射门、关键球员）]
[对手分析、战术特点]
[模型结论、空间值、🚌状态]
预测：[选项]。比分倾向 [比分1] / [比分2]。
```

### 投注明细表
表头：票号 | 串法 | 组合 | 注数 | 倍数 | 金额
- 组合列用"球队名 方向 × 球队名 方向"格式（如"美国 主胜 × 土耳其 胜/平"）
- **金额列禁止添加任何箭头/标识符**（⬆/⬆⬆等符号禁止出现）
- 复式票注数列写"2注"，倍数写"1倍"，金额写"¥4"
- 加注票注数列写"1注"，倍数写"2倍"，金额写"¥4"
- 复式+加注的相邻两行合并展示，便于对比：
  ```
  3 | 2串1 | 美国主胜 × 土耳其胜/平 | 2注 | 1倍 | ¥4
  4 | 2串1 | 美国主胜 × 土耳其胜   | 1注 | 2倍 | ¥4
  ```
- 合计行写在最后：` | | **合计** | **12注** | | **¥30**`

### 出票校验表
表头：检查项 | 状态
固定检查项：胜平负×让球胜平负混合过关、同一场不混串、最高关数校验、单注≥¥2、无反向下注

### 场景分析表
上面一行：`▎ 预估赔率：[球队名]@[赔率] · [球队名]@[赔率] · ...`
**重要：命中情况列必须用球队名（如"仅美国错""仅摩洛哥错"），不能用选项名（如"仅主胜错"会与多个场次混淆）。**

**复式场的场景分析：** 复式场有2种可能的结果（如土耳其胜/平），分别对应不同的赔率和回报。在场景表中拆开写：
```
| 仅摩洛哥错（土耳其胜） | 3,5,7 | ¥42 | +¥12 |
| 仅摩洛哥错（土耳其平） | 3,5   | ¥60 | +¥30 |
```

表头：命中情况 | 中奖票号 | 总回报 | 利润
行顺序：
- 四场全对（复式场分别写"土耳其胜"和"土耳其平"两种情况）
- 仅各场错（逐个列出，复式场拆开写）
- 中两场：全部组合（4场=6种，5场=10种）
- 中一场或零场

利润列：正数用"+¥X"格式，负数用"-¥X"格式，不要出现浮点数长尾（如¥2.51979999999）

### 结尾
```
▎ ⚠️ 竞彩返奖率71%，理性购彩。本方案基于模型-市场价值分歧，不构成投注建议。
```
